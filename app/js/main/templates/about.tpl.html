Purpose

This is a hypothetical app designed to allow users to log into their Twitter account and send payments to their followers or friends

Tech Stack

A front-end heavy application built on Angular and running on Node and Express. I chose angular as the front end to continue learning and expanding my knowledge of this framework. I have deepened my grasp of its architecture and how to pass data between $scope, controllers, and services. Some of the skills learned or enhanced involve $rootScope broadcasting, storing data in a reusable service, and use of $states.

My choice of node and express was more a matter of convenience - although I have never used these before, it seemed relatively easy and quick to set up. The goal was to focus on heavy front end development

Payments

Payments have not been implemented, however there are several routes we can explore to implement this feature.

API integration

The plan would be to have a user authorize both Twitter and Paypal accounts. Simply put, a user would send a tweet to another user using a specific syntax like "!tweetbucks.pay @godspeedyoo $23". Before posting the tweet, the app listens for this syntax with regex and fires events.

These events would then do several things:

  • Verify that the user is connected with paypal
  • Retrieve variables from proposed tweet: timestamp, amount, recipient
  • Check against user's paypal balance to see if payment is possible (or allow the payment handling to be done by redirecting the user to paypal)
  • Match the recipient's twitter handle to their verified paypal account and send the payment.
  • Verify recipient has received it, and automate posting a reply on the tweet by a verified third party (a bot twitter account) to provide confirmation.

This strategy has some concerns. 1) this relies on users having accounts in two places. 2) The payments can only be made via the app, and not another Twitter client 3) Matching a recipient's twitter handle to their paypal account cannot be done unless the recipient provides that data to the app itself. Venmo API does allow for payments to be made to users without a registered account if an email is provided, however.

Hosting

Heroku was an obvious choice as I have used it before and it is quick to set up. This was also a matter of convenience considering the cost of hosting elsewhere which would take away from development time.

Lessons Learned

Exploring various technologies and choosing the right ones can be quite impactful. For example, I initially started out wanting to build a more robust application with a Rails back end, and realized quickly that I was sinking time into unnecessary work which takes away from the focus of this assignment - front end.

Front end development is a hell lot of fun. So is development in general. I had an insane amount of fun.

Architecture is important: if I had a better plan or knowledge of how to structure my front end code, it would have saved a good amount of time in my refactoring. My refactoring process involved pretty much breaking down my angular code and rebuilding.
